Static Public Member Functions

static void DynamicallyAddMarks ()
 Handles the dynamic display of markers Only displays markers, if the display markers button is active. First clears items from the cluster manager and calls clustering to really clear the displayed markers. Always just adds markers to the clusterManager in the current visible region of the map fragment. If the active user is a service technician then only ChargingStations with a defect are added to the clusterManager. If its a normal user then markers are only added if the zoom level is above 8, because on lower zoom levels the displaying of clusters seems unnecessary. In the end cluster() gets called again to render the clusters and markers.

static void cluster ()
 this gets called if markers change and the render cache needs to be cleared this function creates a completely new cluster renderer and sets it to the cluster manager. This is done so the cache is completely cleared, which for example is necessary if a user reports a defect and the color of the marker needs to be changed. Because the camera doesn't move, the marker render is still in the cache and doesn't even change on a clear() and cluster() on the cluster renderer. This function then calls DynamicallyAddMarks() to proceed with the normal dynamic display of markers.

Detailed Description

this is the main view which holds the maps and the different menu buttons This class calls the LogInActivity and the GetChargingStationsActivity and initializes the GoogleMap. It provides the functionality of the bottom menu bar and a functionality to dynamically add markers on different conditions.
